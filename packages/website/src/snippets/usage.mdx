When bundling packages that also export CSS and other external files, <Anchor href="https://bundlejs.com">bundlejs</Anchor> now checks the gzip/brotli size of these external files, even though it won't output any code. Keep this in mind this isn't a bug, however, if it causes confusion I am willing to change this behaviour.

Treeshaking is available, but not all CDNs support access to each packages package.json so there might be slight package version conflicts. The only verified CDN with access to the package.json is <Anchor href="https://unpkg.com">unpkg.com</Anchor>.

Check the console for error messages and warnings.

You can use custom protocols to specify which CDN the module should come from.

* esm.run:react -> <Anchor href="https://esm.run/react">{`https://esm.run/react`}</Anchor>
* esm.sh:react -> <Anchor href="https://cdn.esm.sh/react">{`https://cdn.esm.sh/react`}</Anchor>
* esm:react -> <Anchor href="https://cdn.esm.sh/react">{`https://cdn.esm.sh/react`}</Anchor>
* skypack:react -> <Anchor href="https://cdn.skypack.dev/react">{`https://cdn.skypack.dev/react`}</Anchor>
* unpkg:react -> <Anchor href="https://unpkg.com/react">{`https://unpkg.com/react`}</Anchor>
* deno:preact -> <Anchor href="https://deno.land/x/preact">{`https://deno.land/x/preact`}</Anchor>
* jsdelivr:react -> <Anchor href="https://cdn.jsdelivr.net/npm/react">{`https://cdn.jsdelivr.net/npm/react`}</Anchor>
* jsdelivr.gh:.../react-dom/index.js -> <Anchor href="https://cdn.jsdelivr.net/gh/facebook/react/packages/react-dom/index.js">{`https://cdn.jsdelivr.net/gh/.../react-dom/index.js`}</Anchor>
* github:.../react/index.js -> <Anchor href="https://raw.githubusercontent.com/facebook/react/main/packages/react/index.js">{`https://raw.githubusercontent.com/.../react/index.js`}</Anchor>

e.g.

```ts
import { toStr } from "skypack:@okikio/animate";
// or
export * from "esm:@okikio/animate";
// or
export { animate } from "https://cdn.skypack.dev/@okikio/animate";
```

\* If an error occurs try using a different CDN, by default **bundlejs** uses <Anchor href="https://unpkg.com">unpkg.com</Anchor> but you can use the others.

\* For some packages an error occurs where the default export is excluded from the treeshaken bundle, the solution for this is to manually include the default export like so,

```ts
export * from "solid-dismiss";
// and
export { default } from "solid-dismiss";
```
